# Compilers-Team3

Compilers-Team3 is a Java project managed with **Maven**.
The project currently integrates the main phases developed for the assignment:

- **Lexical analysis**
- **Syntax analysis**
- **Semantic analysis / SDT verification**
- **Parse tree visualization**

The project is packaged as a `.jar` application, and the main entry point is configured to run:

```text
mx.unam.fi.compilers.g5.team03.main.Main
```

## Project Structure

```text
doc/

src/main/java/mx/unam/fi/compilers/g5/team03/

```

## Components

### Lexer
The lexer reads the input source file and tokenizes the supported subset of C.
Its output is normalized and written to:

```text
doc/tokens/tokens.txt
```

This token file is later used as the input for the parser.

### Parser
The parser is implemented as a **Bottom-Up LALR(1)** parser.
Its responsibilities include:

- Reading the grammar from `doc/grammar/grammar.txt`
- Building the canonical LR(1) collection
- Merging compatible states into an LALR(1) collection
- Constructing the `ACTION` / `GOTO` parse table
- Parsing the token stream generated by the lexer
- Building the parse tree

### Semantic Analysis
After parsing, the project performs semantic validation over the parse tree using the SDT rules defined for the assignment.
The semantic rules document is stored at:

```text
doc/semantic/semantic_rules.txt
```

### Parse Tree Viewer
When parsing succeeds, the program displays the parse tree using a Swing-based viewer.

## Input File

The source program used for testing is:

```text
doc/test/test.c
```

You can modify this file to test different valid or invalid inputs.

## Build and Execution

This project is managed with **Maven**.

### Compile the project
```bash
mvn compile
```

### Generate the package
```bash
mvn package
```

### Run the generated JAR
```bash
java -jar target\Compilers-Team3-1.0-SNAPSHOT.jar
```

## Expected Behavior

If the input is syntactically and semantically valid, the program should print:

```text
Parsing Success!
SDT Verified!
```

and then open the parse tree viewer.

If the input contains syntactic or semantic errors, the program will report them in the console.

## Notes

- The grammar is externalized in `doc/grammar/grammar.txt`
- Semantic rules are documented in `doc/semantic/semantic_rules.txt`
- The project uses Maven for dependency management, compilation, packaging, and execution
