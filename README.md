# Compilers-Team3
At this point, the repository's content focuses on the lexical analyzer.
This repository contains all the code necessary for the analyzer to function.
## How to compile
- To compile, you need the Lexer.jar file. This file contains all the .class files necessary for the lexer to function.

- To compile, navigate to the folder containing the .jar file. From the base directory, run the following command in the terminal:
```
cd mx/unam/fi/compilers/g5/src/main
```
- In this folder you will find the Lexer.jar file, which is used to run the application.
To run the file, the following command is required, it must be entered in the terminal:
```
java -jar Lexer.jar
```
## How to use
- To use this lexer, the important file is "test.c".
This file is recognized by the lexical analyzer and is the source of the code it will analyze.
- The file is located in the following path:
```
cd mx/unam/fi/compilers/g5/doc/test
```
- This is where you'll find the "test.c" file, where you can test different codes and demonstrate the proper functioning of the lexer.
